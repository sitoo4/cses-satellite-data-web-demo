# Cluster Reuse Notes

Last updated: 2026-06-05

## Reuse Policy

The web app must reuse existing Cluster processed products as inputs. Formal web plots are regenerated by the backend into the web-owned output tree; old PNG/PDF quicklooks are reference/debug artifacts only.

Primary read-only roots:

```text
/Volumes/Elements/data/idlpython_v2/daily_full
/Volumes/Elements/data/idlpython_v2/daily_compact
/Volumes/Elements/data/idlpython_v2/manifests
/Volumes/Elements/data/idlpython_v2/provenance
```

Reference/debug-only quicklook roots:

```text
/Volumes/Elements/data/idlpython_v2/quicklook_B
/Volumes/Elements/data/idlpython_v2/quicklook_E
/Volumes/Elements/data/idlpython_v2/quicklook_POS
```

Current backend reuse:

- lists processed Cluster dates
- exposes product existence and local paths
- reads `daily_full` NPZ key inventory
- reads `daily_compact` parquet columns
- reads a safe subset of manifest fields
- returns `unsupported` for historical `existing_quicklook_*` requests because those PNGs are not formal web output
- generates formal magnetic, electric, spectrogram, and orbit overview PNGs under `/Volumes/Elements/satellite_data_web/outputs/generated_plots/cluster`
- generates PSD and orbit views from existing `daily_full` arrays without rerunning wavelet or production code
- returns `unavailable` for `cluster_solar_wind_overview` with explicit missing/unverified fields because no validated solar-wind product is exposed in the current processed web datasource

## Product Families

```text
daily_full/YYYY/daily_full_YYYYMMDD.npz
daily_compact/YYYY/daily_compact_YYYYMMDD.parquet
quicklook_B/YYYY/B_YYYYMMDD.png
quicklook_E/YYYY/E_YYYYMMDD.png
quicklook_POS/YYYY/POS_YYYYMMDD.png
manifests/YYYY/manifest_YYYYMMDD.json
provenance/YYYY/daily_provenance_YYYYMMDD.json
```

Manifest paths may contain historical `E:\data\...` strings. The backend uses local path conventions from the configured processed root instead of trusting those strings.

## Boundaries

Do not trigger from web runtime:

- `run_daily_production.py`
- `yearly_dry_run_2005.py`
- monthly dry-run scripts
- IDL `.pro` scripts
- multi-year production
- Figure 3 or formal L-bin/f-L/Cummings matching workflows

Return `blocked` or `unsupported` for:

- formal L-bin conclusions
- formal f-L conclusions
- Figure 3
- final Cummings frequency matching
- modal classification from mixed B/E medians

## Implemented Regenerated Web Plots

Current backend/frontend catalog also includes:

- Cluster time-series subset from `daily_full`
- Cluster export from processed arrays
- Cluster sample-index statistics from processed arrays
- `cluster_magnetic_overview` from existing magnetic arrays
- `cluster_electric_overview` from existing electric arrays
- `cluster_spectrogram_overview` from existing processed PSD arrays
- `cluster_orbit_overview` from existing processed context arrays
- `cluster_solar_wind_overview` unavailable/missing-field response

Future additions should still read existing products as inputs and should not recompute wavelet/PSD/B/E production products unless a separate explicit validation gate is opened. Formal web plot artifacts should remain under `outputs/generated_plots/cluster`.
