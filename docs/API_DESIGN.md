# API Design

Last updated: 2026-06-05

## Purpose

This document defines the first backend API shape for the satellite data web application. The backend should be implemented with Python FastAPI and should keep Cluster and CSES data sources isolated behind a shared datasource interface.

The frontend must never read local raw data files directly. It must call backend API endpoints.

## Principles

- Data-source isolation is mandatory: `cluster` and `cses_hpm` have separate path roots, readers, assumptions, and capability flags.
- All raw-data and processed-data roots come from local config, not hard-coded source files.
- API responses must distinguish `confirmed`, `inferred`, `unsupported`, `todo`, and `blocked`.
- Formal Cluster web plots must be regenerated by the backend from existing processed products and written under `outputs/generated_plots/cluster`; historical quicklook PNGs in `/Volumes/Elements/data/idlpython_v2` are reference/debug inputs only.
- The backend must never copy raw Cluster CDF or CSES H5 files into the web project.
- Long or expensive jobs return a `job_id`; lightweight metadata and plot-catalog listing can return synchronously.

```json
{
```

## Endpoints

### `GET /api/health`

Returns backend status, config-root presence, and basic dependency availability.

### `GET /api/datasources`

Returns all registered datasources and capability flags.

Expected initial datasource names:

- `cluster`
- `cses_hpm`

### `GET /api/datasources/{name}/files`

Lists available files, dates, or products for a datasource.

Cluster query parameters:

- `year`: optional, such as `2005`
- `status`: optional, such as `SUCCESS`
- `product`: optional, one of `daily_full`, `daily_compact`, `quicklook_B`, `quicklook_E`, `quicklook_POS`

Cluster response should use local processed paths and include product existence flags.

CSES query parameters:

- `pattern`: optional file glob, default `**/*.h5`
- `limit`: optional

CSES response should list raw file paths under the configured HPM root, file sizes, and inspection-status flags. It should not expose array contents.

### `GET /api/datasources/{name}/metadata`

Returns datasource-level metadata or metadata for a specified file/date.

Common query parameters:

- `file_id`: optional for CSES
- `date`: optional for Cluster, `YYYYMMDD`

Cluster metadata should include:

- manifest fields after local path normalization
- `daily_full` key inventory
- `daily_compact` column inventory
- quicklook availability
- segment start/end time
- E availability status
- `time_summary` from confirmed `daily_full` `segment_time_context_unix` values. The response includes `status`, `time_variable`, `time_confidence="confirmed"`, `time_units="unix_seconds"`, `sample_count`, UTC `start`/`end`, and `cadence_ms` count/min/median/max.
- `quality_summary` from confirmed `daily_full` `segment_E_quality` values. The response includes `status`, `flag_variable`, `flag_confidence="confirmed"`, `sample_count`, and string-keyed observed-value counts. It does not interpret the flag meanings.

CSES metadata should include:

- H5 tree summary if inspection exists
- dataset path, shape, dtype, attrs
- confirmed/inferred candidate fields
- `time_summary` when the inferred time dataset is mechanically parseable. The response includes `status`, `time_variable`, `time_confidence="inferred"`, `time_units`, `sample_count`, UTC `start`/`end`, and `cadence_ms` count/min/median/max. This is a parsing aid and UI context field, not scientific confirmation of mission time semantics.
- `quality_summary` when the inferred quality flag dataset is readable. The response includes `status`, `flag_variable`, `flag_confidence="inferred"`, `sample_count`, and a string-keyed `distribution`. This exposes observed flag values but does not interpret flag meanings.
- `report_artifact` when the generated inspection `report.md` exists. The frontend links this registered markdown artifact through `/api/artifacts/{artifact_id}` instead of reading filesystem paths directly.

### `GET /api/datasources/{name}/variables`

Returns plottable and exportable variables for a file/date.

Cluster variables should be generated from `daily_full` and `daily_compact` fields. MVP fields:

- B field: `segment_B_GSE`, `segment_B_MFA_after_delete`, `segment_dB_MFA_after_delete`, `segment_dB_MFA_detrended`
- E field: `segment_E_GSE_clean`, `segment_E_corrected_GSE`, `segment_E_MFA`, `segment_dE_MFA_detrended`
- context: `segment_L`, `segment_MLT`, `segment_MLAT`
- PSD: `segment_dB_*_psd`, `segment_dE_*_psd`
- B/E: `segment_BE_poloidal_ratio`, `segment_BE_toroidal_ratio`, `segment_BE_compressional_diagnostic_ratio`

CSES variables should come from inspection output first. If no inspection exists, return `inspection_required`.

### `GET /api/datasources/{name}/plot-catalog`

Returns the datasource-specific plot catalog for the selected file/date. The frontend must use this response rather than hard-coded fixed plot buttons.

Request:

```text
GET /api/datasources/cluster/plot-catalog?file_id=20051203
GET /api/datasources/cses_hpm/plot-catalog?file_id=<relative-h5-path>
```

Response item shape:

```json
{
  "plot_type": "cluster_magnetic_overview",
  "display_name": "Magnetic overview",
  "description": "Regenerated Cluster magnetic combo plot from daily_full fields.",
  "enabled": true,
  "disabled_reason": null,
  "required_fields": ["segment_B_MFA_after_delete", "segment_dB_MFA_detrended"],
  "confirmed_fields": ["segment_B_MFA_after_delete", "segment_dB_MFA_detrended"],
  "inferred_fields": [],
  "output_group": "cluster"
}
```

### `POST /api/datasources/{name}/plot`

Creates or returns a plot.

Request:

```json
{
  "file_id": "20051203",
  "plot_type": "cluster_magnetic_overview",
  "range": {
    "mode": "sample_index",
    "start_index": 0,
    "end_index": 512
  },
  "render": "png"
}
```

Supported Cluster plot types:

- `cluster_magnetic_overview`, regenerated from existing `daily_full` magnetic arrays.
- `cluster_electric_overview`, regenerated from existing `daily_full` electric arrays.
- `cluster_spectrogram_overview`, regenerated from existing `daily_full` B/E PSD arrays without wavelet reruns.
- `cluster_orbit_overview`, regenerated from existing `daily_full` orbit/context arrays.
- `cluster_solar_wind_overview`, returns `status="unavailable"` with explicit `missing_fields` until validated solar-wind/context fields are exposed.

Historical `existing_quicklook_*` requests return `status="unsupported"` because old PNG/PDF quicklooks under `/Volumes/Elements/data/idlpython_v2` are reference/debug only and are not formal web output.

Implemented CSES HPM plot-catalog types after inspection:

- `cses_hpm_magnetic_overview`, showing HPM vector magnetic components and `|B|`.
- `cses_hpm_quality_overview`, showing inferred quality-flag distribution when a flag field exists.
- `cses_hpm_trajectory_overview`, enabled only when latitude/longitude/altitude fields are available.
- `cses_hpm_cadence_overview`, showing mechanically parsed `/UTC_TIME` cadence when parseable.
- `cses_hpm_batch_statistics_overview`, routed to the existing batch-statistics endpoint rather than a PNG plot.
- `cses_hpm_spectrogram_overview`, disabled until time semantics, cadence, magnetic variable, and quality masking are confirmed.

CSES HPM electric-field and solar-wind plot requests return `status="unsupported"` with the magnetometer-only disabled reason.

Successful Phase 6 plot responses also include `job_id`. The job is registered as a synchronous completed job and can be queried through `GET /api/jobs/{job_id}`.

### `POST /api/datasources/{name}/subset`

Returns a cropped data preview and optionally writes a temporary subset artifact.

Request:

```json
{
  "file_id": "20051203",
  "variables": ["segment_B_MFA_after_delete", "segment_MLAT"],
  "range": {
    "mode": "time",
    "start": "2005-12-03T15:00:00Z",
    "end": "2005-12-03T16:00:00Z"
  },
  "preview_limit": 200
}
```

Range modes:

- `time`
- `sample_index`

For Cluster, `time` resolves against confirmed `daily_full` `segment_time_context_unix` values and returns `time_confidence="confirmed"`.
For CSES, `time` is allowed only when the time field is parseable and linked to the chosen variable. Otherwise use `sample_index`.
CSES currently resolves parseable `/UTC_TIME` values mechanically for single-file subset, plot, export, and stats requests. The response keeps `mode="time"` and adds `resolved_mode="sample_index"`, `start_index`, `end_index`, `sample_count`, `time_variable`, and `time_confidence="inferred"`.
Cluster time-range responses use the same shape but `time_variable="segment_time_context_unix"`, `time_units="unix_seconds"`, and `time_confidence="confirmed"`.

### `POST /api/datasources/{name}/timeseries`

Returns bounded sample-aligned variable data for plotting or analysis clients without writing an artifact. This endpoint is the explicit datasource-interface method for "read selected variable timeseries"; it reuses the same bounded range rules as subset/export and never performs an unbounded read.

Request:

```json
{
  "file_id": "CSES_01_HPM_5_L02_A2_sample.h5",
  "variables": ["/B_FGM"],
  "range": {
    "mode": "time",
    "start": "2023-04-19T23:55:44Z",
    "end": "2023-04-19T23:55:47Z",
    "time_variable": "/UTC_TIME"
  }
}
```

Cluster responses return a confirmed UTC axis from `segment_time_context_unix` when the processed time array is present, including sample-index-ranged requests. CSES responses try to parse `/UTC_TIME` mechanically and return `time_axis.kind="utc"` with `confidence="inferred"`; if the time field is unavailable or unparsable, the response falls back to `sample_index` and includes `fallback_reason`.

### `POST /api/datasources/{name}/export`

Writes a bounded export into `outputs/exports/` and returns registered data and manifest artifacts. Cluster export supports sample-index ranges and confirmed `segment_time_context_unix` ranges. CSES export supports sample-index ranges and single-file inferred `/UTC_TIME` ranges when `/UTC_TIME` is mechanically parseable.

Request:

```json
{
  "file_id": "20051203",
  "variables": ["segment_B_MFA_after_delete"],
  "range": {
    "mode": "sample_index",
    "start_index": 0,
    "end_index": 100
  },
  "format": "csv"
}
```

Supported Phase 6 formats:

- `csv`
- `dat`
- `h5`

Reserved:

- `cdf`, returns `unsupported`/reserved until `cdflib` or `spacepy` writer behavior is verified.

Reserved CDF response:

```json
{
  "datasource": "cses_hpm",
  "format": "cdf",
  "status": "unsupported",
  "reserved": true,
  "reason": "Phase 6 supports csv, dat, and h5 export for CSES HPM; CDF remains reserved."
}
```

The frontend displays `CDF TODO` as a disabled export-format option and does not submit CDF export requests from the UI.

Phase 6 response:

```json
{
  "job_id": "export:cluster:<digest>",
  "datasource": "cluster",
  "file_id": "20051203",
  "format": "csv",
  "range": {"mode": "sample_index", "start_index": 0, "end_index": 100, "sample_count": 100},
  "table": {"row_count": 100, "column_count": 5},
  "artifact": {
    "artifact_id": "cluster:export:20051203:<digest>",
    "media_type": "text/csv",
    "exists": true
  },
  "manifest": {
    "datasource": "cluster",
    "file_id": "20051203",
    "original_file": "/Volumes/Elements/data/idlpython_v2/daily_full/2005/daily_full_20051203.npz",
    "export_format": "csv",
    "sample_count": 100
  },
  "manifest_artifact": {
    "artifact_id": "cluster:export_manifest:20051203:<digest>",
    "media_type": "application/json",
    "exists": true
  }
}
```

### `POST /api/datasources/{name}/stats`

Computes bounded statistics for selected variables. Cluster and CSES batch stats use sample-index ranges. CSES single-file stats also accept inferred `/UTC_TIME` ranges when the time vector is parseable and monotonic.

Phase 6 request:

```json
{
  "file_id": "sample.h5",
  "variables": ["/B_FGM"],
  "range": {"mode": "sample_index", "start_index": 0, "end_index": 100},
  "save_format": "json"
}
```

`save_format` is optional. Supported values are `json`, `csv`, `dat`, and `h5`. When present, the backend writes the statistics summary under `outputs/stats/{datasource}/` and returns a registered `stats_artifact`. `cdf` is reserved and returns `unsupported` with `reserved=true`. The frontend defaults to JSON but exposes a stats-format selector for JSON, CSV, DAT, and H5 on both interactive single-file Stats and selected multi-file CSES Batch Stats.

Phase 6 response includes per-variable component stats:

```json
{
  "job_id": "stats:cses_hpm:<digest>",
  "datasource": "cses_hpm",
  "file_id": "sample.h5",
  "save_format": "json",
  "range": {"mode": "sample_index", "start_index": 0, "end_index": 100, "sample_count": 100},
  "variables": [
    {
      "path": "/B_FGM",
      "component_count": 3,
      "stats": [
        {"component": 0, "count": 100, "finite_count": 100, "min": 1.0, "max": 5.0, "mean": 3.0, "std": 1.2}
      ]
    }
  ],
  "stats_artifact": {
    "artifact_id": "cses_hpm:stats:sample_h5:<digest>",
    "media_type": "application/json",
    "exists": true
  }
}
```

Batch CSES request:

```json
{
  "file_ids": ["a.h5", "b.h5"],
  "variables": ["/B_FGM"],
  "time_variable": "/UTC_TIME",
  "flag_variable": "/FLAG_MT",
  "position_variables": {"lat": "/GEO_LAT", "lon": "/GEO_LON", "alt": "/ALTITUDE"},
  "range": {"mode": "sample_index", "start_index": 0, "end_index": 1000}
}
```

The CSES frontend includes a batch file selector next to bounded preview controls. The current selected H5 file is always included; checked additional H5 files are sent as `file_ids` with `/UTC_TIME`, `/FLAG_MT`, `/GEO_LAT`, `/GEO_LON`, and `/ALTITUDE` helper variables when available. Batch stats and batch plot remain sample-index ranged in the current MVP.

Batch CSES response adds:

- `raw_sample_count`, `sample_count`, and `duplicate_count`
- numeric `time_range` and cadence min/max/median
- per-component min/max/mean/median/std and missing ratio
- `vector_magnitude` stats for 3-component variables
- quality flag distribution and spatial coverage when requested
- optional `stats_artifact` when `save_format` is supplied

Batch CSES plot request:

```json
{
  "plot_type": "batch_magnetic_timeseries",
  "file_ids": ["a.h5", "b.h5"],
  "variables": ["/B_FGM"],
  "time_variable": "/UTC_TIME",
  "range": {"mode": "sample_index", "start_index": 0, "end_index": 1000}
}
```

Batch plot response adds `raw_sample_count`, `sample_count`, `duplicate_count`, `time_range`, and a PNG `artifact`. It uses the same numeric time sort/deduplicate behavior as Batch Stats and does not enable CSES spectrogram generation.

Cluster stats for MVP:

- Use existing `daily_full` products.
- Do not launch multi-year production.
- Do not compute formal L-bin/f-L conclusions.
- Optional stats saving uses json/csv/dat/h5 artifacts under `outputs/stats/cluster`; CDF remains reserved.

### `GET /api/jobs/{job_id}`

Returns status for registered processing jobs. In the current MVP, plot, export, and stats endpoints run synchronously and register a completed in-memory job before returning. The returned API payload includes `job_id`.

Job states:

- `queued` (reserved for future async workers)
- `running` (reserved for future async workers)
- `completed`
- `failed`
- `cancelled`

Current completed-job response:

```json
{
  "job_id": "plot:cses_hpm:<digest>",
  "status": "completed",
  "kind": "plot",
  "datasource": "cses_hpm",
  "created_at": "2026-06-05T00:00:00+00:00",
  "finished_at": "2026-06-05T00:00:00+00:00",
  "request": {
    "file_id": "sample.h5",
    "plot_type": "magnetic_timeseries"
  },
  "result": {
    "plot_type": "magnetic_timeseries",
    "artifact": {"artifact_id": "cses_hpm:plot:sample_h5:magnetic_timeseries:<digest>"}
  }
}
```

Unknown jobs return:

```json
{
  "detail": {
    "code": "job_not_found",
    "job_id": "missing"
  }
}
```

Job records are process-local and are not persisted across backend restarts yet.

## Static Output Serving

The backend should expose generated artifacts through a controlled route:

- `GET /api/artifacts/{artifact_id}`

The route maps IDs to registered paths under the configured `outputs_root`. Formal web plot artifacts are written under `outputs/generated_plots`; old quicklook roots are not formal plot-output roots. The route must not serve arbitrary filesystem paths.

2. Datasource registry with stub capabilities. Done in phase 3.
3. CSES H5 inspection report listing, metadata, variables, and sample-index subset preview. Done in phase 3.
4. Cluster file/date listing from processed products. Done in phase 4.
5. Cluster metadata/variables from `daily_full` and `daily_compact`. Done in phase 4.
6. Cluster formal generated plot catalog and overview endpoints. Done after phase 6 plot-catalog update.
7. Frontend MVP.
8. Subset/export/stats. Done in phase 6.
9. Synchronous completed-job registry for plot/export/stats. Done after phase 6 API gap review.
